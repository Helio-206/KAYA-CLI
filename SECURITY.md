# Security Policy

## Supported Scope

KAYA CLI is currently a release candidate and research-oriented local-first communication system. Security issues affecting the current codebase are still important and should be reported responsibly.

## How to Report

Please avoid opening a public issue for a security vulnerability before maintainers have had a chance to assess it.

Send a private report with:

- a clear description of the issue
- the affected area or file path if known
- impact assessment
- reproduction steps or proof of concept if available
- any suggested mitigation

If a dedicated private reporting channel is not yet published for the repository, open a minimal private maintainer contact by using the repository profile contact information and avoid posting exploit details publicly.

## What to Expect

Maintainers will aim to:

- acknowledge receipt
- assess severity and reproducibility
- determine whether the issue affects the current release candidate
- coordinate a fix or mitigation before wider disclosure when practical

## Current Security Boundaries

For the current architecture and explicit non-goals, also read [docs/SECURITY.md](docs/SECURITY.md) and [docs/LIMITATIONS.md](docs/LIMITATIONS.md).
