# Code of Conduct

## Our Standard

KAYA CLI welcomes technical discussion, critique, and contribution. We expect participants to be respectful, precise, and constructive.

Examples of expected behavior:

- discuss ideas and code directly without personal attacks
- assume good faith while still being technically rigorous
- give actionable feedback instead of dismissive comments
- be honest about limits, risks, and uncertainty
- help keep the project accessible to new contributors

Examples of unacceptable behavior:

- harassment, intimidation, or threats
- hateful or discriminatory language
- repeated bad-faith disruption of discussions or reviews
- publishing private information without permission
- sexualized language or unwanted advances

## Scope

This code of conduct applies to repository discussions, issues, pull requests, and presentation-related collaboration spaces connected to the project.

## Enforcement

Project maintainers may remove comments, close discussions, or restrict participation when behavior undermines a safe and constructive working environment.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers through the security or repository contact channel and include enough context for review.
